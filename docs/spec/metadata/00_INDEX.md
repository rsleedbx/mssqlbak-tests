# Metadata spec — index

This subdirectory documents the **catalog metadata** recovery layer: non-data
artifacts (DDL, permissions, statistics, plan guides, Query Store) extracted from
a `.bak` file **without** running SQL Server.  It is a peer of the data-type spec
(`../data_types/`) but covers a different axis: schema structure instead of
per-column value layouts.

The spec convention (code-anchored claims, confidence tags, `Gnn` guess IDs)
follows `../00_MASTER.md` §0.

## Verification model

The correctness harness (`mssqlbak-tests/tools/correctness_coverage/`) measures
metadata accuracy by comparing **recovered metadata** from a `.bak` against a
**ground-truth (GT) sidecar** collected from a live SQL Server restore.

### GT sidecar

Each fixture `.bak` has a companion `<name>.bak.metadata.json` file (the sidecar).
It is generated by `register_bak.py`, which restores the database into a
containerised SQL Server instance and runs per-category T-SQL queries against
`sys.*` views to collect the expected values.

### ValidationResult

`metadata_verify.py: ValidationResult` is the return type of every `verify_*()`
comparator:

```python
@dataclass
class ValidationResult:
    category: str
    n_ok:      int        # items that matched exactly
    n_total:   int        # total items in GT
    missing:   list[str]  # GT items absent from recovered set
    extra:     list[str]  # recovered items absent from GT
    mismatched: list[dict[str, Any]]  # items present in both but differing
    error:     str | None
    unscored:  bool       # category skipped (sidecar absent or exception)
```

A category **passes** when `ok` is `True`: `error is None`, `unscored is False`,
and `missing`, `extra`, `mismatched` are all empty.

### Scoping rules

GT queries often join `sys.tables` (base user tables only) or `sys.objects`
(which excludes database-level DDL triggers).  The recovered set is scoped to
match:

- **`base_table_ids`** — set of `object_id` values whose `type='U'` in
  `sysschobjs`.  Indexes, statistics, and object-level extended properties are
  filtered to this set.  Source: `metadata_verify.py: RecoveredMetadata.base_table_ids`.
- **`ddl_trigger_fqns`** — FQN strings of database-scoped DDL triggers
  (`type='TR'`, `pid==0` in `sysschobjs`).  The modules comparator excludes
  these.  Source: `recover_ddl_trigger_object_ids` in `catalog/recover.py`.
