# SQL Server → Lakehouse / Lakebase: Type and DDL Conversion

Type and DDL element mappings from SQL Server `.bak` files through the
intermediate Arrow representation to Lakehouse (Databricks / Unity Catalog /
Delta Lake) and Lakebase (Databricks managed PostgreSQL).

The Arrow column is the in-memory pivot.  Lakehouse stores Arrow types
using Spark SQL equivalents; Lakebase stores them using standard SQL types.

## DDL Element Conversion

What is preserved, converted, or dropped when going from a SQL Server `.bak`
to a Lakehouse Delta table or a Lakebase pg_dump file / directory archive.

### Column-Level DDL

| SQL Server element | Lakehouse | Lakebase (pg_dump) | Notes |
|---|---|---|---|
| Column name | ✅ Preserved | ✅ Preserved | |
| Data type | ✅ Converted | ✅ Converted | See type tables above |
| `NOT NULL` | ✅ Preserved | ✅ Preserved | Emitted in `CREATE TABLE` when column is non-nullable |
| `NULL` (explicit) | ✅ Preserved | ✅ (column is nullable by default) | |
| `IDENTITY(seed,inc)` | ⚠️ Values extracted as integers | ✅ Emitted | Decoded from `syscolpars.idtval`. Lakebase output: `GENERATED BY DEFAULT AS IDENTITY (START WITH seed INCREMENT BY inc)` in `CREATE TABLE`. |
| `DEFAULT` expression | ⚠️ Dropped | ⚠️ Dropped | Expression body is in system LOB tables, not data pages. |
| `COLLATE` | ⚠️ Dropped | ⚠️ Dropped | Collation IDs are decoded for string routing but not translated to `COLLATE` clauses. |
| Sparse column (`SPARSE`) | ✅ Data preserved | ✅ Data preserved | Values decoded from the sparse vector and written as normal columns. `SPARSE` hint not emitted. |
| Column set (`FOR XML`) | ✅ Data preserved | ✅ Data preserved | Synthesised as a regular XML/text column. |
| Computed column (persisted) | ✅ Values extracted | ✅ Values extracted | Stored as a regular column with its computed value; `AS (expr) PERSISTED` not emitted. |
| Computed column (non-persisted) | — Skipped | — Skipped | No physical storage; omitted from output. |
| Always Encrypted (ciphertext) | ⚠️ Ciphertext bytes | ⚠️ `bytea` ciphertext | Returned as raw bytes; decryption requires the column master key. |

### Table-Level DDL

| SQL Server element | Lakehouse | Lakebase (pg_dump) | Notes |
|---|---|---|---|
| `PRIMARY KEY` | ⚠️ Dropped | ✅ Emitted with original name | `CONSTRAINT "PK_name" PRIMARY KEY (col1, col2)` in `CREATE TABLE`. Lakehouse has no enforced PK; Unity Catalog informational PKs require a post-load `ALTER TABLE`. |
| `UNIQUE` constraint | ⚠️ Dropped | ✅ Emitted with original name | `CONSTRAINT "UQ_name" UNIQUE (col)` in `CREATE TABLE`. |
| `UNIQUE` index (non-constraint) | ⚠️ Dropped | ✅ Emitted (unnamed) | `UNIQUE (col)` in `CREATE TABLE`. |
| Non-unique index | ⚠️ Dropped | ⚠️ Dropped | Not recovered into DDL. Rebuild with `CREATE INDEX` post-restore. |
| `FOREIGN KEY` | ⚠️ Dropped | ✅ Emitted | Column mappings decoded from `syssingleobjrefs`. Emitted as `CONSTRAINT "name" FOREIGN KEY (cols) REFERENCES "schema"."table" (cols)` in `CREATE TABLE`. |
| `CHECK` constraint | ⚠️ Dropped | ⚠️ Dropped | Constraint name only; expression body is not stored on data pages. |
| `CLUSTERED` / `NONCLUSTERED` | — N/A | — N/A | Storage-engine detail; no equivalent in Lakehouse or standard Lakebase DDL. |
| `ROW` / `PAGE` compression | — N/A | — N/A | SQL Server storage compression; data is decoded to plain values. |
| `COLUMNSTORE` | — N/A | — N/A | Decoded to plain rows; columnstore is a storage format, not a schema element. |
| `TEMPORAL` (system-time) | ✅ Both tables extracted | ✅ Both tables extracted | Current and history tables extracted separately as regular tables. `PERIOD FOR SYSTEM_TIME` and versioning metadata dropped. |
| Graph `NODE` / `EDGE` | ✅ Columns extracted | ✅ Columns extracted | `$node_id`, `$from_id`, `$to_id` synthesised as text columns. `AS NODE` / `AS EDGE` clause dropped. |
| Schema (owner) | ✅ `dbo.table` | ✅ `dbo.table` (from `.bak`) or original schema (from pg_dump) | Schema preserved in qualified table name. |
| `FILESTREAM` | 🚫 Out of scope | 🚫 Out of scope | Requires instance-level feature unavailable on Linux. |

### Table Storage and Partitioning

| SQL Server element | Lakehouse | Lakebase (pg_dump) | Notes |
|---|---|---|---|
| `PARTITION FUNCTION` / `PARTITION SCHEME` | ⚠️ Dropped | ⚠️ Dropped | Partitioned tables are extracted as flat tables; all partitions merged into a single output. Partition function and scheme objects not reproduced. |
| `PARTITION BY RANGE` (table) | ⚠️ Dropped | ⚠️ Dropped | Data is fully extracted but the partition clause is not emitted. Rebuild declarative partitioning manually after restore. |
| `FILLFACTOR` | — N/A | ⚠️ Dropped | Index storage detail; not emitted. |
| `WITH (DATA_COMPRESSION=...)` | — N/A | — N/A | Decoded to plain values; see compression rows above. |
| `LEDGER` table | ✅ User columns extracted | ✅ User columns extracted | Hidden ledger columns (`ledger_start_transaction_id`, etc.) exposed as regular columns. `LEDGER = ON` clause not emitted. |
| Memory-optimised (`MEMORY_OPTIMIZED=ON`) | ⚠️ Not extracted | ⚠️ Not extracted | In-memory OLTP tables are not recoverable from a standard `.bak`; only disk-based pages are decoded. |

### Database-Level Objects

These objects have no physical row data in the data pages that mssqlbak reads.
They are never extracted.

| SQL Server object | Lakehouse | Lakebase (pg_dump) | Notes |
|---|---|---|---|
| `VIEW` | 🚫 Not extracted | 🚫 Not extracted | View definitions are in system LOB tables, not data pages. Recreate after restore. |
| `MATERIALIZED VIEW` (indexed view) | 🚫 Not extracted | 🚫 Not extracted | Underlying base-table data is extracted; the indexed view itself is not. Recreate post-restore. |
| `STORED PROCEDURE` | 🚫 Not extracted | 🚫 Not extracted | Procedure bodies live in system LOB tables. |
| `FUNCTION` (scalar / table-valued) | 🚫 Not extracted | 🚫 Not extracted | Same as stored procedures. |
| `TRIGGER` | 🚫 Not extracted | 🚫 Not extracted | Trigger bodies live in system LOB tables. |
| `SEQUENCE` | 🚫 Not extracted | 🚫 Not extracted | Sequence objects have no data pages. Identity column seed/increment is preserved (see above); standalone `CREATE SEQUENCE` objects are not. |
| `SYNONYM` | 🚫 Not extracted | 🚫 Not extracted | |
| `TYPE` (user-defined / CLR) | 🚫 Not extracted | 🚫 Not extracted | UDT columns are decoded to their underlying base type at extraction time. |
| `CLR ASSEMBLY` | 🚫 Not extracted | 🚫 Not extracted | CLR assembly binaries live in system tables. CLR-typed columns are decoded to their base type. |
| `XML SCHEMA COLLECTION` | 🚫 Not extracted | 🚫 Not extracted | Schema validation is dropped; `xml` columns are extracted as plain text. |
| `SCHEMA` (non-dbo) | ✅ `schema.table` | ✅ `schema.table` | Non-dbo schemas are preserved in the qualified table name. |
| `ROLE` / `USER` / `LOGIN` | 🚫 Not extracted | 🚫 Not extracted | Security principals are not in data pages. On Lakehouse, tables are owned by the Unity Catalog principal (user or service principal) running `extract_bak`. On Lakebase, tables are owned by the role that executes `psql` or `pg_restore`. Grant permissions manually after restore. |
| Row-level security (`SECURITY POLICY`) | 🚫 Not extracted | 🚫 Not extracted | Predicates live in system tables; extracted data is unfiltered. |
| Column-level security (`DENY`) | 🚫 Not extracted | 🚫 Not extracted | All columns extracted regardless of column-level permissions. |
| Full-text catalog / index | 🚫 Not extracted | 🚫 Not extracted | Rebuild with `CREATE FULLTEXT INDEX` (Lakebase: `CREATE INDEX … USING GIN`) post-restore. |
| Statistics objects | ⚠️ Dropped | ⚠️ Dropped | Auto-created and manual statistics are not in data pages. Both targets rebuild statistics automatically on first query or via `ANALYZE`. |
| `SERVICE BROKER` (queues, services, contracts) | 🚫 Not extracted | 🚫 Not extracted | Messaging infrastructure objects; no equivalent in Lakehouse or Lakebase. |
| `CERTIFICATE` / `ASYMMETRIC KEY` / `SYMMETRIC KEY` | 🚫 Not extracted | 🚫 Not extracted | Encryption key objects; not in data pages. |
| `LINKED SERVER` | 🚫 Not extracted | 🚫 Not extracted | Remote server references; no equivalent concept. |
| `PLAN GUIDE` | 🚫 Not extracted | 🚫 Not extracted | Query plan hints; no equivalent concept. |
| `DATABASE AUDIT SPECIFICATION` | 🚫 Not extracted | 🚫 Not extracted | Configure audit separately on the target platform. |
| Extended properties | 🚫 Not extracted | 🚫 Not extracted | Stored in `sys.extended_properties`; not in data pages. |
| `DATABASE SCOPED CONFIGURATION` | — N/A | — N/A | Instance/database-level settings; no equivalent. |

### Lakebase-Specific Storage Options (DDL Before Load)

The pg_dump plain text (`.sql`) and directory outputs are files you edit
before calling `psql` or `pg_restore`.  Common post-generation DDL changes:

| What to add | Lakebase syntax | When needed |
|---|---|---|
| TOAST compression (PG 14+) | `ALTER TABLE t ALTER COLUMN c SET COMPRESSION lz4;` | For large `text` / `bytea` columns |
| Column storage type | `ALTER TABLE t ALTER COLUMN c SET STORAGE EXTERNAL;` | Disable TOAST for frequently-scanned columns |
| UUID native type | `ALTER TABLE t ALTER COLUMN c TYPE uuid USING c::uuid;` | `uniqueidentifier` columns |
| PostGIS geometry | `ALTER TABLE t ALTER COLUMN c TYPE geometry USING ST_GeomFromText(c);` | PostGIS is pre-installed on Lakebase — no extension install needed. |
| `time` native type | `ALTER TABLE t ALTER COLUMN c TYPE time USING c::time;` | `time(n)` columns (stored as `text`) |
| Change identity type | `ALTER TABLE t ALTER COLUMN id DROP IDENTITY; ALTER TABLE t ALTER COLUMN id ADD GENERATED ALWAYS AS IDENTITY;` | Change `BY DEFAULT` to `ALWAYS` if you want strict identity enforcement |
| Grant permissions | `GRANT SELECT, INSERT, UPDATE, DELETE ON t TO role;` | Users/roles are not extracted from the `.bak`; grant manually. |

### Lakehouse-Specific Options (Post-Load)

| What to add | Lakehouse syntax | When needed |
|---|---|---|
| Unity Catalog PK (informational) | `ALTER TABLE t ADD CONSTRAINT pk PRIMARY KEY (col) NOT ENFORCED;` | Lakehouse does not enforce PK uniqueness; PK is informational only. |
| Unity Catalog UNIQUE (informational) | `ALTER TABLE t ADD CONSTRAINT uq UNIQUE (col) NOT ENFORCED;` | Same — informational only. |
| Spatial SQL geometry | `ST_GeomFromText(col, srid)` inline in queries, or `ALTER TABLE t ADD COLUMN geom GEOMETRY; UPDATE t SET geom = ST_GeomFromText(wkt_col, 4326);` | Databricks Spatial SQL (80+ functions). WKT strings in `STRING` columns work directly with all `ST_*` functions at query time — no schema change required. |
| Grant permissions | `GRANT SELECT ON t TO \`user@domain\`;` or via Unity Catalog UI | Users/roles are not extracted from the `.bak`; grant via Unity Catalog. |

For the directory format (`pg_restore`), Lakebase DDL changes must be applied
**after** `pg_restore` completes since the binary `toc.dat` does not expose
an editable DDL hook before load.  For the plain text format, edit the `.sql`
file directly before running `psql`.

## Key Notes

- **No unsigned integers in Lakehouse / Lakebase.**  SQL Server `tinyint` (0–255) is
  widened to `int16` / `SMALLINT` so that the maximum value (255) round-trips without
  overflow.

- **`time` loses its native type across both targets.**  Lakehouse has no time-of-day
  column type, so values are stored as ISO 8601 strings.  Lakebase has `time`, but
  since the pivot format is already a string, the column lands as `text` unless the
  DDL is manually adjusted after restore.

- **Timezone handling.**  `datetimeoffset` is converted to UTC before storage.  All
  other datetime types are treated as timezone-naïve.

- **`decimal` precision.**  SQL Server supports up to `decimal(38,s)`; Arrow
  `decimal128` also supports up to 38 digits, so precision is fully preserved.

- **`money` / `smallmoney`.**  Mapped to `DECIMAL` with scale 4 to preserve the
  four decimal places. `money` stores 8 bytes (range ±922 trillion), modelled as
  `decimal128(19,4)`.

- **Spatial types** (`geometry`, `geography`) are extracted as OGC WKT strings.
  Both targets have first-class spatial support:
  - **Lakebase**: PostGIS is pre-installed. Cast with
    `ALTER TABLE t ALTER COLUMN c TYPE geometry USING ST_GeomFromText(c);`
  - **Lakehouse**: Databricks Spatial SQL provides 80+ functions (`ST_Distance`,
    `ST_Intersects`, `ST_Contains`, …) and native `GEOMETRY`/`GEOGRAPHY` column
    types. Convert WKT at query time with `ST_GeomFromText(col, srid)` or store
    natively with a schema change. `STRING` columns are immediately usable with
    all Spatial SQL functions without any migration.

- **`uniqueidentifier`.**  Stored as a hyphenated UUID string.  In Lakebase, cast the
  `text` column to `uuid` with `ALTER TABLE … ALTER COLUMN … TYPE uuid USING col::uuid`.

- **Ownership.**  Users, roles, and logins are not extracted from the `.bak`.  On
  Lakehouse, all created tables are owned by the Unity Catalog principal running
  `extract_bak`.  On Lakebase, tables are owned by the role running `psql` /
  `pg_restore`.  Use `REASSIGN OWNED BY old_role TO new_role;` on Lakebase or Unity
  Catalog grants on Lakehouse to redistribute ownership and access after restore.
