Columnstore indexes have evolved from a read-only data warehouse feature in SQL Server 2012 into a fully updateable, high-performance engine for hybrid transactional and analytical processing (HTAP) [cite: 1.1.8]. They reduce I/O and optimize memory by utilizing aggressive column-wise data compression [cite: 1.1.1].The evolution and optimizations of SQL Server columnstore technology, release by release:1. SQL Server 2012: The IntroductionColumnar Storage: Data stored column-by-column rather than row-by-row, drastically reducing I/O for large analytical queries [cite: 1.1.3].Batch Mode Execution: Allowed the processing of multiple rows (typically batches of 900) simultaneously, significantly lowering CPU overhead [cite: 1.2.5].Limitation: The index was completely read-only [cite: 1.1.10].2. SQL Server 2014: Updateability & New StructureClustered Columnstore Indexes (CCI): The table itself could now be structured as a columnstore, turning the index into the table and eliminating the need for traditional row-based structures [cite: 1.1.4].Delta Store: Allowed for updates and inserts by using a temporary B-tree (rowstore) to catch new data before it was compressed into the main columnstore [cite: 1.1.6].3. SQL Server 2016: Operational Analytics & FlexibilityUpdateable Nonclustered Columnstore Indexes (NCCI): Allowed non-clustered columnstore indexes to be placed on existing row-store tables, facilitating Real-Time Operational Analytics [cite: 1.1.10].Real-Time Analytics Overlap: Updatable NCCIs meant users could run fast OLAP queries on the columnstore while simultaneously performing OLTP updates/inserts on the same table via the B-tree [cite: 1.2.5].Compression Delay: Delayed the compression of delta store rows, preventing locking/latching performance degradation on highly active transactional tables [cite: 1.3.1].Pushdowns: Added String Predicate Pushdown (speeds up querying string data types) and Aggregate Pushdown (computes math operations during table scans) [cite: 1.3.1].4. SQL Server 2017 & 2019: Compute & ConcurrencyMemory Grant Feedback: Evaluated memory requirements dynamically for batch mode operations to prevent spills.Batch Mode on Memory-Optimized Tables: Enabled columnstore analytics on In-Memory OLTP tables, merging high-speed writes with in-memory analytic queries.5. SQL Server 2022: Better Execution & SortingOrdered Columnstore Indexes: Data is physically sorted by a key column prior to compression [cite: 1.2.10]. This dramatically improves compression ratios and allows SQL Server to skip irrelevant rowgroups entirely (segment elimination) when applying filters [cite: 1.2.10].Batch Mode on Rowstore: Batch mode execution—previously exclusive to columnstore indexes—was extended to traditional B-tree structures [cite: 1.2.10].Batch Mode for SORT & Single-Threaded Queries: Sorting operations now leverage batch mode, and analytic operations can use batch mode even in single-threaded queries [cite: 1.2.10].Availability Groups: Improved capabilities to offload analytical workloads to readable secondary replicas [cite: 1.2.10].6. SQL Server 2025: Advanced Enterprise MaintenanceOnline Index Rebuilds: Operations for clustered and nonclustered ordered columnstore indexes can be rebuilt online [cite: 1.1.7].Smarter Database Shrink: Database and file shrink operations can properly move columns of data utilizing large object (LOB) data types like VARCHAR(MAX) [cite: 1.1.7].Better Auto-Elimination: Automated enhancements on ordered columnstore structures help avoid the need to restructure tables for segment elimination [cite: 1.1.7].
